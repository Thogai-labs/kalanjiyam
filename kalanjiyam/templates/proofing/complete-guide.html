{% endmacro %} {% block title %}{{ _('Complete guide to proofing') }} | Kalanjiyam{% endblock %} {% block sidebar %}{{ m.main_nav('complete', current_user=current_user) }}{% endblock %} {% block content %} {{ m.title_and_subtitle( _("Complete guide"), _("Comprehensive guidelines for proofing a text. When in doubt: don't change what the page says!")) }}

This document defines the proofreading guidelines we use on Kalanjiyam. Generally, these guidelines apply to all of the projects we proof.

{{ _table_of_contents(contents) }} {{ _h('what-is-our-goal') }}

We want to create machine-readable texts that we can display on Kalanjiyam and share with others.

We should create these texts at the highest quality we are capable of. We should establish credibility by making clear where these texts come from (which books, editions, editors, publishers, …), how they were created, and who was involved.

{{ _h('the-golden-rule') }}

{{ _("Don't change what the page says.") }}

{{ _('We follow the conventions of the printed book as closely as we can.') }}

  • {{ _("Follow spelling conventions.") }}

    {{ _("If the book says 'kAryyate', write 'kAryyate' (not 'kAryate').") }}

  • {{ _("Follow script conventions.") }}

    {{ _("If the book uses Devanagari, we use Devanagari.") }}

  • {{ _("Follow word-splitting.") }}

    {{ _("Follow the book's sandhi conventions regardless of personal preference.") }}

  • {{ _("Include everything meaningful.") }}

    {{ _("Include spurious verses and footnotes exactly as they appear.") }}

{{ _h('characters') }} {% call components.div_note() %} Rules that start with [auto] can be fixed with software automatically. Fixing them manually is not a good use of your valuable time. {% endcall %} {{ _h2('characters', 'eol-hyphens') }}

If the printed line ends with a hyphen, we keep the hyphen. Our program uses these hyphens to stitch together different lines:

{% call _example() -%} sa bhagavAnsRSTvedaM jagattasya ca sthitiM cikI- rSuH ... {%- endcall %}

Exception: If the hyphen is part of a word that would otherwise have a hyphen (e.g. "front-end"), join the words together so that our program preserves the hyphen.

{{ _h2('characters', 'eol-dashes') }}

Hyphens (-)

{{ _('Use for end of lines or compounds. Write with a single dash.') }}

Em dashes (—)

{{ _('Used to separate phrases. Write with two dashes (--).') }}

{{ _h2('characters', 'auto-spaces') }}

We don't leave any space between a dash and the words around it:

{{ _compare('arthaH -- eka'|d, 'arthaH--eka'|d) }} {{ _h2('characters', 'auto-quotes') }}

We use "straight quotes" without any special formatting.

{{ _compare('“naraH”'|d, '"naraH"'|d) }} {{ _h('paragraphs') }} {{ _h2('paragraphs', 'para-breaks') }}

Keep line breaks and line-ending hyphens. Proofing is easier when we can quickly compare our digitized text to the original image.

{{ _h2('paragraphs', 'para-blank-lines') }}

Separate paragraphs and verses with blank lines to maintain structural clarity.

{{ _h2('paragraphs', 'para-headers') }}

{{ _('Page headers and page footers (but not footnotes!) should be deleted. They do not contain useful digital information.') }}

{{ _h2('paragraphs', 'para-footnotes') }}

In the main text, mark the character that indicates the footnote by wrapping it in brackets and placing a ^ character after the first bracket.

{{ _compare('naraH1'|d, 'naraH[^1]'|d) }} {{ _h('pages') }}

Skip any content that is not part of the original book. This includes handwritten notes, stamps, watermarks, dirt, stains, etc.

{{ _h('annotations') }} {{ _h2('annotations', 'errors') }}

If the text has a typographical error, wrap the error in the <error> tag. If needed, add a fix with the <fix> tag.
